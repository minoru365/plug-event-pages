はじめての Docker

Docker は、アプリを動かす環境をまとめて扱いやすくする仕組みです。 まずは細かいことを全部覚えなくても大丈夫です。 このページでは、Docker Desktop をインストールして、最初のサンプルを動かすところまでを体験します。

01 · Docker とは

Docker は何をしてくれるもの？

Docker は、アプリとその実行環境をまとめて扱いやすくする仕組みです。 「自分のPCでは動くのに、ほかの人のPCでは動かない」を減らすのに役立ちます。

Docker が役立つ場面

開発に必要な環境をそろえやすくなり、同じ構成でアプリを動かしやすくなります。

最初はここだけでOK

まずは「Docker Desktop を入れて、サンプルを動かす」ところまでできれば十分です。

03 · インストール

Docker Desktop を入れる

1
公式サイトを開く

https://www.docker.com/products/docker-desktop/ を開き、Windows 版をダウンロードします。

2
インストーラーを実行する

画面の案内に沿ってインストールします。 途中で必要な設定が表示された場合も、まずは案内に沿って進めれば大丈夫です。

3
Docker Desktop を起動する

インストール後に Docker Desktop を起動し、準備が完了するまで待ちます。

補足

Windows では、Docker Desktop の利用にあたって WSL2 が必要になることがあります。 ここでは詳細は追わず、まずはインストーラーの案内に沿って進めれば大丈夫です。

04 · 動作確認

Docker が使えるか確認する

PowerShell またはコマンドプロンプトを開いて、次のコマンドを入力します。

# Docker のバージョン確認
docker --version

確認ポイント

バージョン番号が表示されれば、Docker コマンドが使える状態です。 エラーになる場合は、Docker Desktop が起動しているかを確認してみてください。

05 · はじめての実行

hello-world を動かしてみる

次のコマンドを実行すると、Docker がサンプル用のイメージを取得して、短いメッセージを表示します。 

# 最初のサンプルを実行
docker run hello-world

ここで何が起きているの？

  • 必要なイメージを取得する
  • そのイメージを使ってコンテナを起動する
  • メッセージを表示して終了する
06 · よく出る言葉

最初に知っておくと楽な用語

イメージ

アプリを動かすためのひな形です。

コンテナ

イメージをもとに、実際に動いているものです。

Docker Desktop

Windows や Mac で Docker を使いやすくするアプリです。

08 · 次のステップ

慣れてきたらやってみること

入門では、まず Docker Desktop を入れて docker run hello-world を試すところまでできれば十分です。 慣れてきたら、次は自分のアプリを Docker で動かす流れに進みます。

1
Dockerfile を作る

アプリを動かすための「作り方」を書くファイルです。Python スクリプトなら数行で書けます。

2
docker build でイメージを作る

Dockerfile をもとに、実行用のひな形であるイメージを作ります。

3
docker run でコンテナを起動する

作成したイメージから、実際に動くコンテナを起動します。

4
docker compose を使う

Web アプリとデータベースなど、複数のサービスをまとめて扱いやすくします。

次のステップのイメージ

自分のアプリを Docker で動かすときは、だいたい次の流れになります。

flowchart LR A["Dockerfile\n作り方を書く"] -->|docker build| B["イメージ\n実行用のひな形"] B -->|docker run| C["コンテナ\n実際に動く"]
💡 Python スクリプトを Docker で動かす最小例
まずはこの2ファイルを同じフォルダに作って試してみましょう。 
# Dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY hello.py .
CMD ["python", "hello.py"]
# hello.py
print("Hello from Docker!")
# ビルドして実行
docker build -t my-hello .
docker run my-hello
# → Hello from Docker!

困ったときは

  • Docker Desktop が起動しているか確認する（タスクトレイのクジラアイコン）
  • エラーメッセージをそのままコピーして検索すると解決策が見つかることが多いです
  • PLUG の Discord で気軽に質問してください
